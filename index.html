sublime 连接ftp服务器

一、打开sublime，用Ctrl+Shift+P打开控制面板，输入install命令，下载名为SFTP的插件

二、下载完之后，可新建一个可存放远程代码的文件夹，拖入sublime的project中，右键找到SFTP/FTP—> Map to remote, 次数会弹出一个json文件，这个文件需要配置一些信息,其中需要配置的项如下：

  • host
  • user
  • password
  • port
  • remote_path: 需要在远程ftp新建一个属于你自己的文件夹,这里填的就是该文件夹的路径

三、保存刚刚的json配置文件之后，右键project中的文件夹，找到SFTP/FTP—>download folder，即可下载ftp服务器的代码

四、修改代码然后保存，代码会自动更新到ftp中属于你的那个模块

Angular与外界通讯,$http服务

代码展示

基本了解

  • $http服务简单的封装了浏览器原生的XMLHttpRequest对象
  • $http接收一个对象参数，这里面包含了HTTP请求的配置内容
  • 返回状态码
  • 返回的promise对象上有三个常用方法，then(), success(), error();then()会接收到完整的相应对象，而success和error则已经对返回的对象进行解析分离
  • then()方法接收了四个相应对象
    data 返回的json数据
    status 相应的http状态码,在200和299之间认为响应成功，success函数会被调用，否则error会被调用
    headers 请求发送的HTTP头信息
    config 原始请求的完整设置对象

快捷方法

$http服务提供了一些方法可以简化复杂设置。这些快捷方法包括：

  • $http.get 两个参数

  • $http.post 三个参数

  • 还有其他常用的请求类型包括
    DELETE, PUT, JSONP

$http 的详细配置项

  • method 包括 GET,DELETE, POST, PUT, JSONP,等等
  • url 请求地址
  • params 可以是字符串或对象，会被转换成查询字符串追加在URL后面
  • data 通常在发送POST请求时使用
  • cache 布尔型，如果设置为true，那么angularJS会用默认的$http缓存来对GET请求进行缓存

缓存http请求

  • 默认情况下，$http服务不会对请求进行本地缓存，但如果将$http的配置项cache设置为true的话，可以启动缓存机制。在第一次发送请求时，$http服务会向指定url发送请求，第二次发送同一个请求时，$http会从缓存中取回请求结果。
  • cache不一定要设置为true，根据自己项目的逻辑需求而定

伪元素:after和:before

::before 和 ::after是css中的比较常用的伪元素

一、为什么使用伪元素？

::after和::before伪元素定义和存在于css中，就说明其必定为页面的样式做出了一定的贡献。它不是真的html元素，只是在css中定义的，插入到html中的，是html页面更美观的元素而已。html失去了伪元素还是一份完整的html文档，只是失去了该伪元素定义的那份美观而已。
一句话，伪元素不会影响文档流。

二、:after和::after及浏览器兼容性

  • ::after是一种新语法，是为了区别伪类(:)和伪元素(::)而创建的
  • 只要是支持双冒号(::)语法的浏览器就支持单冒号(:)
  • IE8 只支持单冒号(:), 建议使用:after 一获得大多数浏览器的支持
  • IE8 以下不支持伪元素

三、:after和:before的基本用法

  1. css选择器:before

  2. 若content为空，可以把它当成一个盒子来使用。默认是内联元素，必须定义成块 级元素才能定义宽高

  3. :before是插入到当前选择器的其他内容的前面，而不是当前选择器的前面
    可发现，插入的伪元素的内容不是在text-container外面，而是在里面，插入到text-container里面的其他内容的前面。如果非要插入到text-container前面，增加一个div即可实现

  4. 属性content的值不仅可以是文本内容，还可以是图片
    可是下面的方法很难操控图片，无法改变图片大小，所以更加推荐将伪元素作为一个div来使用，然后使用background来定义背景

  5. 使用伪元素也可以很方面的使用字体图标，可以自定义大小和颜色
    需引入多种格式的字体文件以兼容多数浏览器

  6. content的值还可以使用attr()函数来将相应属性的值转换成字符串
    attr()可以得到特定属性的值并把它作为插入的文本构成一个伪元素

一些浏览器不能兼容的伪元素的属性

  • FireFox 3.5- 不支持伪元素的绝对定位
  • IE8 不支持伪元素的z-index

需注意

伪元素不会出现在DOM中。这些元素不是真正的元素。考虑到网页的可用性，可访问性，特别是考虑到SEO，不能用伪元素来显示网页的主要内容
